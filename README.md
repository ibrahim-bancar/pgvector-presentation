# Semantic Similarity Search in PostgreSQL Using pgvector

## What is this tool?
pgvector is an open-source PostgreSQL extension for storing and querying vector embeddings inside PostgreSQL. It enables exact and approximate nearest-neighbor search directly in the database, which makes it useful for semantic search, recommendation systems, and similarity-based retrieval.

## Installation
Install the Python dependency:

    python -m pip install -r requirements.txt

## Running the Example
Start the PostgreSQL container:

    docker compose down -v
    docker compose up -d

Wait until PostgreSQL is ready:

    until docker exec pgvector-db pg_isready -U postgres >/dev/null 2>&1; do
      echo "PostgreSQL starting..."
      sleep 2
    done

Initialize the extension, table, and sample data:

    docker exec -i pgvector-db psql -U postgres -d pgvectordb -f /sql/01_init.sql

Run the query demonstrations:

    docker exec -i pgvector-db psql -U postgres -d pgvectordb -f /sql/02_queries.sql

Create and inspect the HNSW index:

    docker exec -i pgvector-db psql -U postgres -d pgvectordb -f /sql/03_index_demo.sql

Run the Python demo:

    python app/demo.py

## Demo Design
This demo uses one table named `documents`.

Columns:
- `id`: unique row identifier
- `title`: document title
- `category`: document category
- `content`: short document text
- `embedding`: 4-dimensional vector representation stored as `vector(4)`

The demo includes 14 handcrafted sample documents across multiple categories:
- database
- ai
- devops
- orchestration
- analytics
- mlops

The embeddings in this demo are manually designed sample vectors. They are not generated by a real embedding model. Their purpose is to simulate semantic similarity in a simple and understandable way for presentation.

## Query Logic
The SQL demo shows:
- full table listing
- database-oriented nearest-neighbor search
- AI-oriented nearest-neighbor search
- DevOps-oriented nearest-neighbor search
- MLOps-oriented nearest-neighbor search
- HNSW index creation for faster nearest-neighbor search

Similarity is measured with L2 distance using the `<->` operator.

## Technical Summary
This project uses:
- PostgreSQL with the pgvector extension
- a `vector(4)` column for embeddings
- L2 distance search with the `<->` operator
- HNSW indexing with `vector_l2_ops`
- Docker for reproducible setup
- Python with `psycopg` for database access
